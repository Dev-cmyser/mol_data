Концепция чем-то похожая на lisp

У нас есть некое абстрактное выражение:

- Константа - выражение результат (значение) которого задается вручную явно
- Переменная - содержит результат другого выражения
- Функция - принимает на вход несколько выражений и возвращает результат

Например, значением переменной может быть константа, функция или другая переменная.

Аналогично, в функцию можно передать в качестве аргументов объявленные ранее константы, переменные или описать их тут же в процессе.

В начале, мы описываем глобальные переменные. Это могут быть значения, которые нужно вычислить или набор исходных данных от которых будут зависеть значения других вычисляемых переменных.

Глобальные переменные доступны на всех уронях в любых выражениях.

Для переменной или функции можно объявить вложенные переменные - они будут доступны уже только для вложенных выражений, но не для выражений объявленных выше.

Всё работает на $mol_book2. Первый разворот - список глобальных переменных.

Кликая на одну из глобальных перменных, отрывается редактор выражения этой перменной на следующем развороте и так далее в глубину по мере построения выражения.
